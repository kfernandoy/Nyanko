<objective>
Orchestrate the wave-1 modules into the live startup sequence and expose the OBS-01 logs action to the UI. This plan adds the splash window (D-01), rewires `index.ts` into an orchestrated startup gate (splash → spawn → readiness → main window, with a dev short-circuit and before-quit kill), establishes the first `ipcMain.handle` in the repo (`ipc.ts`), extends the preload `nyanko` bridge, and adds the "open logs folder" button to the existing `DetectorSettingsView` "acerca" tab.

Purpose: This is the visible payoff of Phase 2 — it kills the old "Cargando biblioteca ~1min" by moving readiness into the main process (D-02), guarantees no orphan sidecars on quit (D-08), and gives the user a real logs action (OBS-01). It depends on Plan 01's `sidecar.ts` and `logging.ts`.
Output: `splash.ts`, `ipc.ts`, an orchestrated `index.ts`, the extended preload, and the logs button + i18n keys.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → main (IPC) | the sandboxed renderer and the splash invoke main handlers via the contextBridge; only whitelisted channels are exposed |
| splash → main (IPC) | pre-renderer window triggers retry/quit/openLogs — control only, no data path |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-IPC | Tampering/Elevation | ipc.ts openLogsFolder handler | medium | mitigate | The handler takes NO renderer argument and opens the fixed app.getPath('logs') — no attacker-controlled path reaches shell.openPath. retryStartup/quit are control-only (no payload). |
| T-02-PRE | Elevation | preload nyanko bridge | high | mitigate | Only three named functions are exposed via contextBridge over ipcRenderer.invoke/send; raw ipcRenderer/Node is never exposed (sandbox:true preserved). |
| T-02-SPLASH | Elevation | splash.ts window | high | mitigate | Splash reuses the identical secure webPreferences (contextIsolation/nodeIntegration:false/sandbox/webSecurity) and the same hardened preload — no privileged window is introduced (CLAUDE.md constraint). |
| T-02-ORPH | Denial of Service | index.ts before-quit | medium | mitigate | killSidecar registered on before-quit guarantees the tracked child tree is killed on every exit path (D-08); no orphan nyanko-api.exe. |
</threat_model>
