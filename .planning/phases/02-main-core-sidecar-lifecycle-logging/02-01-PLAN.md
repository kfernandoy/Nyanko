<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| main → OS process spawn | main launches the bundled nyanko-api.exe; an attacker-controlled exe path or PATH resolution would run arbitrary code |
| sidecar → filesystem (port file) | main reads the `<userData>/port` file written by the sidecar; untrusted-ish local content parsed into an int |
| main → localhost HTTP | readiness probe hits 127.0.0.1:<port>/api/health — localhost only, never a remote host |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-INJ | Tampering/Elevation | sidecar.ts spawnSidecar | high | mitigate | Spawn the exe by ABSOLUTE resolved path (resolveSidecarExe → resources or explicit NYANKO_SIDECAR_EXE override), `shell:false`, never PATH-resolved; the exe is the bundled sidecar, not a name lookup. |
| T-02-PORT | Tampering | sidecar.ts parsePortFile | low | mitigate | Parse is trim+parseInt with null on NaN; the port only feeds a fixed 127.0.0.1 URL, never a host — a poisoned port file can at worst point at another localhost port and fail the health check. |
| T-02-HOST | Information Disclosure | sidecar.ts healthUrl | low | mitigate | URL host is hard-coded 127.0.0.1; only the port is variable. No remote egress. |
| T-02-ORPH | Denial of Service | sidecar.ts killSidecar | medium | mitigate | Single reusable killSidecar() with graceful→`taskkill /T /F` on the tracked pid kills the PyInstaller child tree (D-07); called from both exit paths in Plan 02 (D-08) so no orphans on any path. |
| T-02-SC | Tampering | electron-log install | high | mitigate | Only ONE new dependency (electron-log) is added; verify it on npmjs.com before install (legitimate, widely-used Electron logging lib). No other packages introduced. |
</threat_model>
