  <files>apps/desktop/electron/main/compat-paths.test.ts, apps/desktop/package.json</files>

  <behavior>
    - `userDataDir("C:\\Users\\x\\AppData\\Roaming")` returns a path ending in `app.nyanko.desktop`.
    - `assertUserDataDir("C:\\Users\\x\\AppData\\Roaming\\app.nyanko.desktop")` does NOT throw.
    - `assertUserDataDir("C:\\Users\\x\\AppData\\Roaming\\Nyanko")` throws (the exact orphaned-library failure mode the guard exists to prevent).
  </behavior>
  <action>
    Create `apps/desktop/electron/main/compat-paths.test.ts` using Node's built-in `node:test` + `node:assert` (no test framework dependency). Import from `./compat-paths` and cover the three behaviors above. Because compat-paths is electron-free, this runs under plain Node — no electron mock, no app boot.
    Add a `"test:datadir"` script to package.json that runs the file with tsx type-stripping: `node --import tsx --test electron/main/compat-paths.test.ts`.
    Keep it to this one guard module; do not test window creation or electron APIs here.
  </action>
  <verify>
    <automated>cd apps/desktop && npm run test:datadir</automated>
  </verify>
  <acceptance_criteria>
    `npm run test:datadir` passes: the wrong-path case asserts a throw, the correct-path case asserts no throw, and userDataDir is asserted to end in `app.nyanko.desktop`.
  </acceptance_criteria>
  <done>compat-paths.test.ts covers the three behaviors with node:test/node:assert; `test:datadir` script added; `npm run test:datadir` passes.</done>
