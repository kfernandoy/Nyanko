<key_findings>
Grounding facts discovered while planning — the executor must honor these:

1. The renderer runs UNCHANGED under Electron. `src/App.tsx` renders its custom titlebar only behind `HAS_TAURI = "__TAURI_INTERNALS__" in window` (line 1054), and every other native call (`api.ts`, `autostart.ts`, `discord.ts`, `windowPrefs.ts`) is guarded by the same check. With no Tauri internals the app degrades gracefully. Do NOT set `__TAURI_INTERNALS__` in the preload.
2. CORS constraint (load-bearing): `apps/backend/nyanko_api/config.py` `allowed_origins` includes `desktop_url = "http://localhost:1420"`. The renderer dev server MUST stay on port 1420 (strictPort), or fetches to the :8765 backend are blocked and the library never loads. The existing `vite.config.ts` already pins 1420 — reuse it.
3. `api.ts` with no Tauri internals resolves the API to `http://127.0.0.1:8765` (DEFAULT_API_URL). That matches the manual dev backend. Do NOT set `VITE_API_URL` for the electron-vite dev flow — that would repoint the renderer away from the manual backend.
4. Consequence of finding 1: in Phase 1 the frameless window has no functional titlebar (HAS_TAURI is false, so the custom titlebar is not rendered, and frame:false removes the native one). This is expected — window controls/titlebar wiring is Phase 4 (NATIVE-04). Close the dev window via the OS (Alt+F4) or devtools. Do NOT add titlebar logic here.
</key_findings>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → repo | New devDependencies (electron, electron-vite, tsx) enter the build via install |
| renderer → main | Untrusted web content reaches the main process only through the preload contextBridge |
| renderer → backend | Renderer fetches the local Python API over HTTP (127.0.0.1:8765) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-SC | Tampering | npm installs (electron, electron-vite, tsx) | high | mitigate | Task 1 blocking-human legitimacy checkpoint verifies each package on npmjs.com before install |
| T-01-01 | Elevation of Privilege | renderer webPreferences | high | mitigate | contextIsolation:true, nodeIntegration:false, sandbox:true, webSecurity:true set verbatim in Task 3; preload exposes no Node |
| T-01-02 | Tampering | userData path resolution | high | mitigate | setPath(userData) before any path access + assertUserDataDir crash-guard (Task 3), unit-verified in Plan 02 |
| T-01-03 | Information Disclosure | preload surface | medium | mitigate | Minimal preload exposes only a placeholder namespace; no fs/child_process/ipc-wildcard reaches the renderer |
</threat_model>
