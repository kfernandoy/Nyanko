<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer -> main (IPC) | Renderer sets window prefs via window-prefs:set |
| main -> sidecar (HTTP) | Tray toggles detection via local HTTP POST |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-04 | Tampering | window-prefs.ts fs write | high | mitigate | Path is `join(app.getPath("userData"), "window_prefs.json")` — never renderer input; write is scoped to the app data dir only, no path traversal possible |
| T-04-05 | Tampering | window-prefs:set payload | medium | mitigate | Coerce payload to exactly three booleans and drop unknown keys before persisting — no prototype pollution / arbitrary JSON write |
| T-04-06 | Spoofing / SSRF | tray detection POST | medium | mitigate | URL is fixed to `http://127.0.0.1:{port}` with port from the local `port` file (fallback 8765); no renderer-supplied host/port; 5s timeout |
| T-04-07 | Tampering | preload bridge | medium | mitigate | Only named typed methods getWindowPrefs/setWindowPrefs exposed; no raw ipcRenderer |
</threat_model>
